# 折线统计图 — 教学参考

## 拆到公理

**核心**：折线统计图的本质是**用点标记数据，用连线展示变化趋势**。它解决的核心问题是：不仅看到"是多少"，更要看到"怎么变的"——上升、下降、平稳、突变。

**不可再拆的公理**：
1. 横轴 = 时间/类别（自变量），纵轴 = 数量（因变量）
2. 一个点 = 一个时刻/类别的数据值
3. 连线 = 两点间的变化趋势（陡 = 变化快，平 = 变化慢）
4. 趋势 ≠ 精确值——折线只连接已知数据点，两点之间是估算

## 前置依赖

- 数据收集
- 条形统计图（作为对比——条形图看"量"，折线图看"变"）
- 数对和坐标（坐标系基础）
- 可能性（数据趋势与预测的关系）

## 常见误区 Top 3

1. **把连线当实际数据**：以为折线上每一点都代表实际数据（实际只有标记点才是真实数据），连线只是趋势辅助线。
2. **忽略纵轴刻度起点**：纵轴不从0开始时，变化幅度被放大——学生可能把小幅波动误读为剧烈变化。
3. **混淆上升速度和上升量**：折线陡 = 变化速度快，但不等于变化量大。

## 交互实验室方案

**line-chart-interact 组件**：
- 初始折线图：某城市一周气温（3个数据点：周一15°C、周三20°C、周日18°C）
- 拖拽数据点：
  - 把周三从20°C拖到25°C → 折线自动重绘，连线变陡
  - 把周日从18°C拖到15°C → 下降趋势更明显
- 添加数据点：在周二位置点击 → 新增一个点，输入17°C → 折线更新
- 纵轴缩放：切换"0起点"和"10起点" → 观察同一数据在两种刻度下的视觉差异
- 关键：拖拽操作让学生**用手感受**数据变化如何影响趋势，而非被动看图

## 对抗挑战 4 道

1. **小明的结论**：看气温折线图，周三到周四折线下降很陡，小明说"周四气温暴跌了10度"。但实际上周三25°C，周四22°C，只降了3度。为什么折线看起来"跌了很多"？

2. **找反例**："折线图只能用来展示变化趋势，不能展示具体数值。"这句话对吗？什么时候折线图能精确读取数值，什么时候不能？

3. **识别陷阱**：一张股价折线图从 50 元涨到 55 元，纵轴从 50 开始（不是 0），折线看起来翻了一倍。小华大喊"股价翻倍了！"她错在哪？

4. **逆向思考**：一段折线：水平 → 缓慢上升 → 陡然下降 → 水平。描述这四段分别对应什么实际情况？给出一个可能的真实场景。

## 费曼填空 4 道

1. "折线统计图和条形统计图最大的区别是：条形图看______________________，折线图看______________________。"
2. "折线越陡，说明变化越____（快/慢）；折线越平，说明变化越____（快/慢）；折线水平，说明______________________。"
3. "看折线图时，要注意纵轴是不是从____开始的。如果不是，变化的视觉效果会被______________________。"
4. "折线上两个数据点之间的线段告诉我们的是______________________，但不是每一个点都代表______________________。"

## 分层练习 3 组

**组1 直接应用**：根据数据绘制折线图：小明1-6月身高：120, 122, 125, 128, 131, 135(cm)。哪两个月增长最快？

**组2 变式迁移**：某商店上半年销售额折线图中，纵轴从80万开始。小明说"上半年销售额增长了3倍"。为什么这可能是个错误判断？如何修正？

**组3 综合应用**：给出四年来的降水量折线图（有明显上升趋势）。预测明年的降水量大概在什么范围？你的预测依据是什么？什么因素可能导致预测不准确？
