## 3. 状态机（线性阶段机）

### 任务状态流转

```
⏳ 空闲 → 🏁 已认领 → 🔧 施工中 → ⏳ 待审查 → ✅ 通过 或 ❌ 需修正(→重试)
     ↑                                                ↓
     └──────────────── 可协商 ──────────────────→ 下一阶段
```

### 认领协议

| 触点 | 触发时机 | 动作 |
|------|---------|------|
| 第一触点 | Ariste 启动 (AGENTS.md step11) | 扫描空闲任务 → 认领 |
| 第二触点 | Cron 15min 审查 | 连续 2 轮无人认领 → 自动分配最高优先级 |
| 第三触点 | Codex 轮询 | 每小时检查 PROGRESS.md，直接认领并标注 |

### 超时规则

| 转变 | 超时 | 动作 |
|------|------|------|
| ⏳ 空闲 → 无人认领 | 4h | 群内提醒 |
| 🏁 已认领 → 无施工 commit | 2h | ⚠️ 施工超时警告 |
| ⏳ 待审查 → 无 cron 响应 | 30min | 手动审查 |
| ❌ 需修正 → 无响应 | 4h | 群内 @Sky |

## 4. 陷阱注册表（持续更新）

| 陷阱 | 症状 | 对策 | 发现 |
|------|------|------|------|
| **认领≠在施工** | 只推状态更新，无实际工作 commit | 超时从认领后首次*工作*commit算，纯状态更新不重置时钟 | 05-16 |
| **任务粒度过大** | "升级6课"是一个大 batch，长时间无中间交付 | 每批 ≤3 课或要求中间检查点 | 05-16 |
| **第一触点失效** | Ariste 长会话中任务创建，启动扫描已过 | cron 第二触点补位（2轮=30min内自动分配） | 05-16 |
| **越界修改** | Codex 改了 scope 之外的文件 | 审查检查 NOT_IN_SCOPE | 来自白皮书 |
| **审计脚本过时** | 新组件不在 audit.js | 每次增结构同步更新 audit | 来自白皮书 |
| **cron 上下文缺失** | lightContext 不记得之前状态 | 每次执行读最新 PROGRESS.md | 来自白皮书 |
